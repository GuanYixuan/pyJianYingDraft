● 根据项目结构，二次开发可以从以下几个方面入手：

  项目架构

  pyJianYingDraft/
  ├── __init__.py          # 公开API导出
  ├── script_file.py       # 核心类 - 草稿创建/素材/轨道/片段管理
  ├── draft_folder.py      # 文件夹管理 - 创建/加载模板/批量操作
  ├── track.py             # 轨道类型和基类
  ├── segment.py           # 片段基类 (ClipSettings, Speed)
  ├── video_segment.py     # 视频片段 (含转场/滤镜/混合模式)
  ├── audio_segment.py     # 音频片段 (含淡入淡出)
  ├── text_segment.py      # 文字片段
  ├── time_util.py         # 时间工具 (SEC=1000000微秒)
  └── metadata/            # 枚举定义 (特效/滤镜/动画/字体)

  二次开发方向

  1. 新增片段类型 - 在 segment.py 定义基类，然后在对应类型文件中实现

  2. 新增特效/滤镜 - 在 metadata/ 目录下添加新的枚举类型

  3. 扩展轨道功能 - 修改 track.py 中的 Track 类

  4. Windows 自动化导出 - jianying_controller.py 使用 UI 自动化控制剪映

  开发命令

  # 安装开发依赖
  pip install -e .

  # 代码检查
  flake8 pyJianYingDraft/

  # 运行演示
  python demo.py

  关键概念

  - 时间格式: 内部使用微秒，用 SEC = 1000000 转换
  - 轨道叠加: 同类型轨道用 relative_index 排序，高值在前
  - 模板模式: 用 DraftFolder.load_template() 加载现有草稿作为模板

  你具体想开发什么功能？我可以给出更具体的建议。

     Context

     用户希望开发一个 SaaS 系统：用户输入文案、图片、视频，结合模板自动生成
     剪映草稿，支持人工微调后下载。

     技术选型已确定：
     - 前端：React
     - 后端：Python (FastAPI + pyJianYingDraft)
     - 数据库：PostgreSQL

     ---
     草稿生成引擎 (draft_engine)

     Generator 类

     复用 pyJianYingDraft 库：

     # backend/app/draft_engine/generator.py
     from pyJianYingDraft import ScriptFile, trange, tim

     class DraftGenerator:
         def generate(
             self,
             template_content: dict,           # 模板 JSON
             materials_mapping: dict,          # 素材映射
             text_replacements: dict,           # 文本替换
             output_settings: dict             # {width, height, fps}
         ) -> dict:
             # 1. 加载模板到 ScriptFile
             script = ScriptFile.load_template_from_dict(template_content)

             # 2. 替换素材
             for template_name, material_path in materials_mapping.items():
                 script.replace_material_by_name(template_name,
     material_path)

             # 3. 替换文本
             for seg_id, new_text in text_replacements.items():
                 script.replace_text(seg_id, new_text)

             # 4. 导出 JSON
             return script.dumps()
